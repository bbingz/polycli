## R2 (runtime) — `review.mjs` housekeeping

Two small-surface items both in `plugins/polycli/scripts/lib/review.mjs`.

### R2a — `readYamlScalar` uses regex for structured keys

**Current state:** `readYamlScalar` regex-parses `api_key` / `api_base` / `model` / `provider` from MiniMax config. Multi-line YAML values, unusual quoting, and indented-block scalars would slip through silently. Flagged in `docs/review-fb64b1e.md:196`.

**Proposed fix shape:** replace the regex helper with a minimal hand-rolled parser that:

- supports `key: value` (plain)
- supports `key: "value"` and `key: 'value'` (quoted, with standard escape handling)
- rejects (throws with an explicit error message) block scalars (`key: |`) and folded scalars (`key: >`) — we don't support those shapes in review config today
- rejects multi-line strings

No YAML library dependency. Keep the parser private to `review.mjs`.

**Test plan:** add tests in `plugins/polycli/scripts/tests/review.test.mjs` that feed:
- plain key
- double-quoted key with embedded spaces
- single-quoted key
- block scalar key (`|`) — assert throws
- malformed line (no colon) — assert throws
- comment line (starts with `#`) — assert ignored

**Scope guard:** no switch to a YAML library. No changes to `buildReviewRuntimeOptions` or `REVIEW_HARD_CONSTRAINTS`. This is just tightening one helper.

### R2b — `writeReviewTempFile` leaves tmp dirs on exit

**Current state:** `writeReviewTempFile` uses `mkdtempSync` but has no cleanup. Long-running hosts accumulate per-review directories in `os.tmpdir()`.

**Proposed fix shape:**

```js
// in review.mjs module scope
const reviewTempDirs = new Set();

process.once("exit", () => {
  for (const dir of reviewTempDirs) {
    try {
      fs.rmSync(dir, { recursive: true, force: true });
    } catch {
      // best-effort; process is exiting
    }
  }
});

// inside writeReviewTempFile, after mkdtempSync:
reviewTempDirs.add(dirPath);
```

**Test plan:** add one test that spawns a child process invoking `writeReviewTempFile`, captures the returned path, waits for the child to exit, and asserts the tmp dir no longer exists.

**Scope guard:** best-effort cleanup only. Do NOT add SIGTERM / SIGINT handlers (those compete with the host's own signal handling). Do NOT add a periodic sweep. Just the `exit` listener.

### Combined delivery

Two fixes in one commit:

fix: review.mjs housekeeping — YAML scalar hardening and tmp cleanup

- readYamlScalar rewritten as a minimal hand-rolled parser that rejects block/folded scalars instead of silently misparsing
- writeReviewTempFile registers created dirs with a process exit listener for best-effort cleanup
